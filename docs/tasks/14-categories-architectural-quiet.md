## Цель
Полностью пересобрать блок категорий в эстетике **Apple/Aesop**: архитектурная чистота, выровненная геометрия, много воздуха, минимум декора. Пользователь явно отказался от предыдущих направлений (журнальная сетка / типографическая лента / polaroid-коллаж) — все три промахнулись по премиум-восприятию.

## Контекст
- Файл: `components/Categories.tsx` (полная замена). Используется только на главной (`app/page.tsx`).
- Данные: массив `categories` из `lib/products.ts` (6 элементов: `slug`, `title`, `subtitle`, `description`, `image`).
- Стек: Tailwind 3.4, Framer Motion 11, `next/image`. Шрифты: `font-display` Playfair, `font-sans` Inter, `font-script` Cormorant.

## Диагностика прошлой попытки (polaroid)
Пользователь явно отметил: «не премиум», «хаотично», «слишком сладко», «скучно». Mood: **архитектурная чистота** — Apple, Aesop. Референса нет.

## Запреты (важно — не повторять прошлых ошибок)
- ❌ Никаких поворотов карточек / `rotate-*`.
- ❌ Никаких перекрытий, абсолютного позиционирования с offset, «брошенных на стол» элементов.
- ❌ Никакого `font-script` / Cormorant italic / рукописных подписей.
- ❌ Никаких журнальных меток («ISSUE 06», «p.01-06», «№ 01», «Collection», «CONTENTS»).
- ❌ Никаких SVG-орнаментов, флоруша, виньеток, разделителей-завитков.
- ❌ Никаких «полароидных» рамок, белых полей вокруг фото, gradient passepartout.
- ❌ Никакого тёмного / розового overlay на фото.
- ❌ Никаких ярких blur-пятен в палитре rose/gold на фоне (если оставлять — максимально приглушённо, либо вовсе убрать).
- ❌ Никакой `noise` / бумажной текстуры.
- ❌ Никакого Marquee, infinite scroll, parallax.
- ❌ Никаких иконок-стрелочек в розовых кругах с поворотом.

## Что должно быть
1. **Идеально выровненная сетка** — 3 колонки × 2 ряда на `lg`, 2 колонки на `md`, 1 колонка на mobile. Все 6 карточек **одинакового размера и формата**.
2. **Большие отступы между карточками** — `gap-12` или больше на `lg`, `gap-8` на `md`. Воздух — это премиум.
3. **Каждая карточка**:
   - Чистое фото, `aspect-[4/5]` (или единый aspect-ratio на всех), `object-cover`, без рамок, без overlay, без скруглений (`rounded-none`) либо очень мелкие (`rounded-sm`).
   - Под фото — отступ 16-20px, затем тонкая линия (`border-t border-ink-200`, opacity 50-60%), затем pt-4.
   - Имя категории — `font-sans` (Inter) medium/regular, `text-base` или `text-lg`, цвет `text-ink-900`. Без `font-display`, без капса (можно начальные капс только если выглядит лучше — но попробуй обычный регистр сначала). Без трекинга.
   - Под именем — одна короткая строка описания/subtitle, `text-sm text-ink-500` (приглушённо). Используй `c.subtitle`.
   - Справа от имени или в правом нижнем углу — мелкий sans `→` или Lucide `ArrowRight` тонкой обводкой, `text-ink-400`. Без круга, без фона.
4. **Заголовок секции** — спокойный, короткий, без декора:
   - Eyebrow «Категории» можно оставить простым `text-xs uppercase tracking-widest text-ink-500`.
   - H2 — короткий, не «Шесть тёплых вселенных», а что-то более сдержанное (например, «Коллекции» или «Категории»). Можно оставить «Шесть тёплых вселенных», но без украшений вокруг.
   - Параграф под h2 — короткий, на одну-две строки максимум, `text-ink-600`.
   - Никаких SVG-флорушей, ISSUE-меток.
5. **Фон секции** — однотонный, очень светлый: `bg-ivory-50` или `bg-white` или `bg-ivory-100`. **Без blur-пятен**. Никакой текстуры. Просто свет.
6. **Hover-эффект — минимальный**:
   - Фото: лёгкая смена opacity (например, 1 → 0.88) ИЛИ тонкое затемнение overlay через `bg-ink-900/5`. **Никакого зума, никакого scale**.
   - Имя категории: цвет → `text-rose-700` (или `text-ink-700` если хочется ещё спокойнее).
   - Стрелка `→`: плавный сдвиг вправо на 4-6px (translateX). 
   - Линия под фото: цвет линии меняется на `border-ink-400`.
   - Duration 0.3-0.4s, ease-out.
7. **Padding секции**: большие — `py-24 lg:py-32`. Воздух сверху и снизу.

## Acceptance criteria
- [x] Все запреты из секции выше соблюдены — нет поворотов, перекрытий, скриптовых шрифтов, журнальных меток, орнаментов, полароидных рамок, overlay, текстур, marquee.
- [x] Сетка 3×2 на lg, 2×3 на md, 1×6 на mobile. Все карточки одного размера.
- [x] Между карточками большие отступы (`gap-x-10 gap-y-16 lg:gap-x-12 lg:gap-y-20` или подобное).
- [x] Фото в едином `aspect-[4/5]` (или другой выбранный, но единый), `object-cover`, без скруглений или с минимальными.
- [x] Под каждым фото: тонкая линия `border-t border-ink-200`, имя категории в Inter (нормальный регистр), subtitle в `text-sm text-ink-500`, тонкая стрелка `→` справа.
- [x] Hover: opacity фото 1→0.88, цвет имени к `text-rose-700`, стрелка сдвигается +4-6px вправо, линия чуть темнеет. Никакого scale, zoom, rotate.
- [x] Заголовок секции: короткий, без декора. Подходящий eyebrow + h2 + один параграф.
- [x] Фон секции — однотонный светлый (`bg-ivory-50` / `bg-white` / `bg-ivory-100`). Без blur-пятен и текстур.
- [x] Padding секции `py-24 lg:py-32`.
- [x] Анимации входа — спокойные: stagger fade-in + лёгкий slide-up (8-12px), duration 0.5s. Без blur-in, без вращений.
- [x] Доступность: `<Link>` оборачивает всю карточку, `aria-label`, `focus-visible` ring (можно тонкий `ring-1 ring-ink-900 ring-offset-4`), keyboard-фокус активирует hover-эффект.
- [x] **Не запускать `npm run build`** — это засоряет dev-кеш и сломает hot reload. Вместо этого запусти `npx tsc --noEmit` для проверки типов.
