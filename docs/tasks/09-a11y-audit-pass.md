## Цель
Привести проект к базовому соответствию WCAG 2.1 AA: доступность с клавиатуры, читалок, корректный фокус и контраст. Это и про пользователей с особыми потребностями, и про SEO/legal.

## Контекст
- В проекте 19 компонентов и 10 страниц — нужен сплошной проход.
- Внимание к: `Header.tsx`, `Footer.tsx`, `CartDrawer.tsx`, `ProductCard.tsx`, `PersonalizationForm.tsx`, `CartView.tsx`, любые модалки/drawer-ы.
- Палитра rose/ivory — есть риск низкого контраста для `text-ink-400` на `bg-ivory-*`.

## Acceptance criteria
- [ ] Все `<img>` имеют осмысленный `alt` (после задачи 03 это уже `next/image` — проверь, что alt не пустые там где должны быть осмысленные).
- [ ] Все icon-only кнопки/ссылки (бургер, закрытие drawer, иконка корзины, "удалить") имеют `aria-label`.
- [ ] `CartDrawer` имеет `role="dialog"`, `aria-modal="true"`, `aria-labelledby`. Закрывается по Esc. Фокус возвращается на триггер. Внутри — focus trap.
- [ ] В `app/layout.tsx` добавлен skip-link `<a href="#main" className="sr-only focus:not-sr-only ...">Перейти к содержимому</a>`, у `<main>` есть `id="main"`.
- [ ] Все интерактивные элементы имеют видимый `focus-visible` (например, `focus-visible:ring-2 focus-visible:ring-rose-400 focus-visible:ring-offset-2`).
- [ ] Контраст: проверить пары `text-* на bg-*` где есть подозрение. Заменить слабый текст на более тёмный оттенок.
- [ ] Формы (контакты, checkout, персонализация) — `<label>` связан с `<input>` через `htmlFor`/`id`, ошибки имеют `role="alert"` или `aria-live="polite"`.
- [ ] Заголовки идут в правильной иерархии (h1 → h2 → h3), без скачков.
- [ ] Тестировать с клавиатуры: можно дойти до всех интерактивных элементов Tab'ом, активировать Enter/Space.

## Не входит в задачу
- Полная поддержка прокладок screen reader (тестирование с VoiceOver) — требует ручной верификации пользователем.
- Реализация переключателя темы или большого шрифта.
- Перевод alt-текстов на английский.
