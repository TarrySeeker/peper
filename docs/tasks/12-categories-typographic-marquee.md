## Цель
Полностью переосмыслить блок категорий — уйти от сетки-плиток к редакционной типографической ленте в духе будуара. Каждая категория — большая изящная строка, которая «оживает» при наведении: имя плывёт marquee-лентой и рядом раскрывается фото. Эстетика: романтика-винтаж, орнаменты, тонкая фурнитура.

## Контекст
- Файл: `components/Categories.tsx` (заменяется целиком). Используется только на главной (`app/page.tsx`).
- Данные: массив `categories` из `lib/products.ts` (6 элементов: `slug`, `title`, `subtitle`, `description`, `image`).
- Стек: Tailwind 3.4 (палитра `rose / gold / ivory / ink`), Framer Motion 11, `next/image`, Lucide.
- Существующий компонент `components/Marquee.tsx` есть в проекте — посмотри, можно ли переиспользовать вместо своего marquee.
- Шрифты доступны: `font-display` (Playfair Display), `font-sans` (Inter), `font-script` (Cormorant Garamond italic).

## Решения интервью
1. **Концепция** — типографическая лента: 6 категорий → 6 строк-«полос». Каждая строка содержит большое имя категории и (раскрывающееся) фото-превью.
2. **Доминанта** — баланс: типографика ведёт, но раскрытое фото равно по силе. Перед раскрытием — типографика, после — двухчастная композиция.
3. **Hover-эффект** — marquee + раскрытие:
   - При наведении на строку имя категории дублируется и плывёт бесконечной лентой (например, «Крабики · Krabiki · Крабики · ...»).
   - Параллельно рядом (справа или слева, на твоё усмотрение) появляется фото-превью категории в винтажной рамке.
   - Подзаголовок (`subtitle`) показывается в `font-script italic` как подпись под фото или у строки.
4. **Mood** — romantic-vintage: между строками тонкие розовые орнаменты-разделители (можно SVG: завитки, точки-фурнитура, ‧·‧·‧·‧, веточки, виньетки), фото в рамке с тонким золотым/розовым «паспарту», возможные мелкие декоративные элементы в углах секции.

## Acceptance criteria
- [x] Сетка-плитки удалена. Блок построен как **вертикальная лента из 6 строк** (по числу категорий).
- [x] В покое каждая строка содержит: порядковый номер (01/06, 02/06...) мелким капсом + большое имя категории (`font-display` или `font-script`, размер на `lg` ≥ 4rem) + мелкий subtitle в `font-script italic` + тонкая стрелочка/индикатор справа.
- [x] При hover на строке:
  - имя категории дублируется и плывёт бесконечной marquee-лентой (можно через CSS-keyframes или Framer Motion).
  - рядом (в той же строке или с боку) плавно появляется фото категории (`next/image`, `<Image fill>` в обёртке с aspect-ratio), в винтажной рамке (тонкий гольденный/розовый бордер + лёгкое внутреннее «паспарту»).
  - другие строки слегка приглушаются (opacity ~0.4–0.5) для усиления фокуса.
- [x] Между строками — тонкий romantic-vintage разделитель (SVG-орнамент: завиток/точки-фурнитура/звёздочка/веточка в палитре `rose-300`/`gold-400`).
- [x] В углах секции (опц.) — мелкие декоративные элементы (виньетки, орнаментальные уголки). Не перегружать.
- [x] Mobile-адаптив (< md): на mobile marquee-эффект может быть упрощён (т.к. hover ограничен): фото показывается всегда или по тапу, имя категории — крупное на всю ширину, орнаментальные разделители тоньше. Не должно ломать UX.
- [x] Анимации входа (Framer Motion) изящные: stagger по строкам, blur-in/fade-in, разделители проявляются после имён.
- [x] Доступность: каждая строка — `<Link href="/catalog/<slug>">`, фокус с клавиатуры активирует тот же визуальный эффект что hover (через `:focus-within`), `aria-label` с именем категории, декор `aria-hidden="true"`.
- [x] `npm run build` проходит. Никаких новых пакетов не добавлять.
- [x] Фоновый декор (розовые blur-пятна) предыдущей версии можно сохранить или адаптировать под romantic-vintage mood — на твоё усмотрение, но фон секции не должен быть пустым.
